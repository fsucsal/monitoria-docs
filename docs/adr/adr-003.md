# ADR-003 — Banco de Dados Principal: PostgreSQL

**Status:** Aprovado

## Contexto

O sistema será desenvolvido em um contexto acadêmico, utilizando:

- Backend em Spring
- Frontend em React
- Arquitetura de Monólito Modular

O domínio da aplicação exige **forte integridade relacional** entre entidades como, por exemplo:

- Usuários
- Perfis de acesso
- Cursos, turmas e disciplinas
- Inscrições e vínculos
- Registros operacionais e históricos

Além disso, o projeto precisa:

- Ser **simples de manter**
- Ter **baixo custo operacional**
- Ser **fácil de ensinar para desenvolvedores iniciantes**
- Manter **consistência e confiabilidade dos dados**
- Permitir **evolução futura**, se necessário

## Decisão

Utilizar o **:contentReference[oaicite:0]{index=0}** como **banco de dados relacional principal** do projeto, seguindo os princípios:

- ✅ Modelo de dados **relacional e normalizado**
- ✅ Uso de **chaves estrangeiras** para garantir integridade
- ✅ Transações ACID para garantir consistência
- ✅ Uso de migrations para controle de versão do banco
- ✅ Um único banco principal no início do projeto

O PostgreSQL será tratado como a **fonte oficial da verdade dos dados** da aplicação.

## Alternativas Consideradas

- MySQL
- :contentReference[oaicite:1]{index=1}
- SQL Server
- Uso combinado de múltiplos bancos desde o início

Essas alternativas foram descartadas por:

- Aumentarem a complexidade operacional
- Dificultarem o ensino de modelagem relacional
- Não trazerem benefícios claros para o escopo atual do projeto

## Consequências

### Positivas

- ✅ Excelente suporte a relacionamentos complexos
- ✅ Forte integridade referencial
- ✅ Confiabilidade
