# ADR-006 — Containerização: Docker como padrão de execução

**Status:** Aprovado

## Contexto

O sistema será desenvolvido por uma equipe acadêmica, com desenvolvedores utilizando diferentes sistemas operacionais e ambientes locais.

Além disso:

- Não há um profissional dedicado de DevOps
- Haverá apenas **DevOps em formação**
- O projeto utilizará:
  - Backend em Spring
  - Frontend em React
  - Banco de dados PostgreSQL
- É necessário garantir que:
  - O ambiente funcione igualmente para todos
  - O projeto seja fácil de subir
  - Problemas de “funciona na minha máquina” sejam evitados
  - O onboarding de novos membros seja rápido

## Decisão

Padronizar o uso do **:contentReference[oaicite:0]{index=0}** como tecnologia oficial de **containerização do projeto**, adotando os seguintes princípios:

- ✅ Toda a aplicação deverá rodar em containers
- ✅ O banco de dados deverá rodar em container
- ✅ O frontend poderá rodar:
  - Em container (preferencialmente)
  - Ou localmente em modo de desenvolvimento, se necessário
- ✅ Será utilizado **docker-compose** para orquestrar:
  - Backend
  - Frontend
  - Banco de dados
- ✅ O ambiente Docker será tratado como:
  - Padrão oficial de desenvolvimento
  - Base para homologação
  - Base para produção

O Docker será a **fonte oficial de verdade do ambiente de execução da aplicação**.

## Alternativas Consideradas

- Execução direta no host
- Máquinas virtuais tradicionais
- Ambientes manuais sem padronização

Essas alternativas foram descartadas por:

- Gerarem inconsistências entre ambientes
- Dificultarem o onboarding
- Aumentarem o risco de erros de configuração
- Exigirem mais manutenção manual

## Consequências

### Positivas

- ✅ Ambiente totalmente reproduzível
- ✅ Redução drástica de problemas de ambiente
- ✅ Facilita a automação de build e deploy no futuro
- ✅ Excelente ferramenta educacional para o time
- ✅ Padronização entre todos os desenvolvedores
- ✅ Base pronta para futura implantação de CI/CD

### Negativas

- ❌ Curva de aprendizado inicial para os alunos
- ❌ Dependência de máquinas minimamente capazes de rodar containers
- ❌ Possível aumento no consumo de recursos locais

## Diretrizes de Uso

- O repositório deverá conter:

  - `Dockerfile` do backend
  - `Dockerfile` do frontend (quando aplicável)
  - `docker-compose.yml` para execução completa do sistema

- Nenhum desenvolvedor deverá:

  - Subir o sistema em produção fora do ambiente Docker
  - Definir dependências apenas no ambiente local

- A dupla de **DevOps em formação**
