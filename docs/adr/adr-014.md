# ADR-014 — Estratégia de evolução arquitetural: extração futura para microsserviços

**Status:** Aprovado (Planejado)

## Contexto

O sistema será inicialmente desenvolvido como um **Monólito Modular**, visando:

- Redução de complexidade inicial
- Facilidade de desenvolvimento para uma equipe acadêmica
- Menor custo operacional no início do projeto

Entretanto, já foi identificado que alguns módulos possuem características que podem exigir **escalabilidade, independência de deploy e reutilização por outros sistemas**, especialmente:

- Geração de certificados (já assíncrona)
- Relatórios
- Autenticação

Além disso, existe a previsão de integração direta com um **Sistema de Eventos**, que também deverá consumir funcionalidades de certificação.

---

## Decisão

O sistema será desenvolvido inicialmente como **Monólito Modular**, porém com **diretriz arquitetural obrigatória de permitir extração futura para microsserviços** dos seguintes módulos:

- `certification`
- `reports`
- `identity`

Diretrizes obrigatórias de preparo:

- Separação rigorosa por módulos (packages claramente isolados)
- Regras de negócio concentradas no domínio do módulo
- Dependência apenas de interfaces entre módulos
- Banco de dados com limites claros de responsabilidade por domínio
- Comunicação futura preparada para REST/mensageria
- Processos assíncronos já previstos (ex: certificados)

A migração para microsserviços **não ocorrerá no escopo inicial do projeto**, sendo considerada apenas quando houver:

- Alta demanda de escala
- Integração com múltiplos sistemas externos
- Gargalos claros de performance no monólito

---

## Alternativas Consideradas

- Permanecer em monólito definitivo sem preparo para extração
- Iniciar o projeto diretamente em arquitetura de microsserviços

---

## Consequências

- ✅ Arquitetura preparada para crescimento futuro
- ✅ Evita reescrita completa dos módulos críticos
- ✅ Facilita reutilização entre sistemas (ex: eventos)
- ✅ Mantém baixa complexidade no início do projeto
- ✅ Alinhado com a geração assíncrona de certificados
- ❌ Exige maior disciplina na separação de dependências
- ❌ Pode limitar algumas decisões rápidas de curto prazo
