# Padrões da Engenharia — Guia Oficial

Este diretório reúne todos os padrões operacionais, técnicos e organizacionais adotados pelo time de engenharia.

O objetivo é garantir consistência, qualidade, previsibilidade e aceleração de onboarding para novos membros.

Cada padrão aqui documentado deve:

- Ser simples e direto.
- Indicar como fazer, quando usar e por quê.
- Incluir exemplos reais sempre que possível.
- Ser revisado periodicamente conforme o sistema evolui.

**Seções**

**1. Padrões de Git & Fluxo de Desenvolvimento**

Define como o time trabalha no dia a dia com Git:

- Nomeação de branches
- Estrutura do fluxo (GitHub Flow, GitLab Flow, Trunk, etc.)
- Regras de PR/MR
- Política de commits
- Estratégia de versionamento
- Estratégias de merge (squash, rebase, merge commit)
